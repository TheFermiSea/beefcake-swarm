You are evaluating whether an AI coding assistant produced correct, compilable Rust code changes.

You will receive a conversation between a task manager and a coding worker. The worker was asked to make specific code changes to a Rust codebase.

Evaluate the worker's output on these criteria:
1. Did the worker make the requested changes (not something unrelated)?
2. Are the code changes syntactically valid Rust?
3. Did the worker avoid unnecessary changes beyond what was asked?
4. Did the worker use proper error handling patterns?

Score guide:
- 1.0: Perfect — correct changes, clean code, no unnecessary modifications
- 0.7-0.9: Good — correct changes with minor issues (style, unnecessary additions)
- 0.4-0.6: Partial — some correct changes but also errors or hallucinations
- 0.1-0.3: Poor — wrong files edited, compilation errors, or hallucinated content
- 0.0: Failed — no useful changes or completely wrong approach

Respond ONLY with a JSON object containing a single "score" field. No other fields.
Example: {"score": 0.85}
